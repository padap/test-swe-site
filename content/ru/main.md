# Введение

Стремительное развитие больших языковых моделей (LLM) в области разработки программного обеспечения выявило критические ограничения существующих бенчмарков, в частности широко используемого датасета SWE-bench. Недавние исследования обнаружили серьезные проблемы с загрязнением данных, где значительная часть успешных патчей включает прямую утечку решений или проходит из-за недостаточных тестов. **SWE-MERA** решает эти фундаментальные проблемы через автоматизированный сбор реальных задач из GitHub и строгую валидацию качества.

Сложность реальной разработки программного обеспечения охватывает агентов для кодирования и широкий спектр задач text-to-code. Хотя SWE-bench был создан из GitHub issues и соответствующих pull request'ов, его **статичная природа** приводит к двум основным проблемам: *утечка данных* (модели запоминают решения) и *насыщение бенчмарка* (потеря эффективности при достижении моделями почти идеальных результатов).

SWE-MERA внедряет **динамические квартальные обновления** для обеспечения: (1) актуальности — задачи отражают последние вызовы в разработке ПО; (2) честной оценки — модели тестируются на свежих задачах, минимизируя утечку данных; (3) непрерывного улучшения — бенчмарк эволюционирует вместе с достижениями в ИИ и инженерных практиках.

# Методология

## Семиэтапный конвейер сбора данных

SWE-MERA реализует надежный семиэтапный конвейер, эффективно обеспечивающий качество и минимизирующий риски загрязнения:

1. **Отбор репозиториев:** Python-репозитории с 10+ звездами, 10+ форками, недавней активностью и open-source лицензиями
2. **Построение маппинга PR-Issue:** Однозначное соответствие между merged pull request'ами и закрытыми issues
3. **Извлечение и фильтрация метаданных:** Парсинг метаданных issue и PR с фильтрацией по длине описания
4. **Извлечение и валидация патчей:** Валидация git diff'ов для обеспечения модификации как исходного кода, так и тестов
5. **Валидация сборки репозитория:** Docker-based настройка окружения с верификацией выполнения pytest
6. **End-to-End тестирование задач:** Тестирование в контролируемой среде для воспроизводимости
7. **LLM-based оценка качества:** Модель Qwen3-32B оценивает корректность задачи, корректность тестов, полноту тестов и сложность

## Статистика датасета

- **Масштаб:** Идентифицировано приблизительно 10,000 потенциальных задач
- **Текущая доступность:** 728 валидированных образцов из 200+ репозиториев
- **Частота обновлений:** Квартальные обновления с новыми реальными задачами
- **Среда выполнения:** Docker-based контейнеры со стандартизированными базовыми образами
- **Контроль качества:** Многоэтапная фильтрация и LLM-based валидация

Весь конвейер реализован как [Python-пакет](https://pypi.org/project/repositorytest) и может быть выполнен для любого GitHub-репозитория, обеспечивая воспроизводимость и расширяемость.

# Ключевые особенности

## Динамические обновления

В отличие от статических бенчмарков, SWE-MERA обновляется ежеквартально новыми, ранее не встречавшимися задачами. Этот непрерывный цикл обновления:
- Отражает последние вызовы в разработке программного обеспечения
- Минимизирует риски запоминания моделями решений
- Поддерживает различительную способность бенчмарка со временем
- Позволяет обнаруживать загрязнение данных через временной анализ

## Валидация качества

Каждая задача проходит строгую валидацию:
- **Автоматизированное тестирование:** pytest-based верификация в изолированных Docker-окружениях
- **LLM-based фильтрация:** Задачи из нижнего квартиля отфильтровываются на основе оценок корректности и полноты
- **Реальная релевантность:** Сохраняются только задачи с реальными решениями и всесторонними тестами
- **Баланс сложности:** Сохраняются как простые, так и сложные задачи для комплексной оценки

## Детектирование загрязнения данных

[Интерактивный лидерборд](#/leaderboard) оснащен временным слайдером, позволяющим пользователям визуализировать метрики оценки в разные временные периоды, идентифицировать потенциальные события загрязнения в обучающих данных моделей, сравнивать производительность на задачах до и после cutoff, и обеспечивать прозрачную и воспроизводимую оценку.

# Оценка

SWE-MERA демонстрирует сильную различительную способность на современных моделях. Оценка с использованием агента Aider на дюжине недавних LLM показывает четкую стратификацию производительности. Бенчмарк использует две ключевые метрики: pass@1 (процент успеха с первой попытки) и pass@6 (процент успеха среди шести независимых попыток).

Сравнение год к году выявляет интересные паттерны: некоторые модели показывают различия в производительности между задачами 2024 и 2025 годов, что потенциально указывает на разную степень экспозиции к данным.

# Процесс подачи результатов

Для участия в оценке SWE-MERA:

1. **Загрузите датасет:** Доступен на [Hugging Face](https://huggingface.co/datasets/MERA-evaluation/SWE-MERA)
2. **Запустите агента:** Выполните ваш агент для разработки ПО на датасете
3. **Отправьте результаты:** Создайте pull request в [репозиторий подач](https://github.com/MERA-Evaluation/SWE-MERA-submissions)
4. **Обновление лидерборда:** Валидные подачи появляются в течение двух рабочих дней
