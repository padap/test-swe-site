# Введение

SWE-bench — текущий стандарт для оценки кодовых агентов на задачах из реальных GitHub-репозиториев. Однако датасет статичен, не обновлялся более двух лет и с высокой вероятностью частично «протёк» в данные обучения моделей (как на этапе pretraining, так и SFT). Это снижает объективность оценки: модели могут воспроизводить запомненные решения вместо реального решения задач.

**SWE-MERA** решает эти проблемы за счёт динамического подхода: задачи автоматически собираются из GitHub и регулярно обновляются. Это позволяет поддерживать актуальность, снижать риск утечки данных и делать оценку более честной.

# Подробности

## Методология

### Семиэтапный конвейер сбора данных

SWE-MERA реализует надежный семиэтапный конвейер, эффективно обеспечивающий качество и минимизирующий риски загрязнения:

1. **Отбор репозиториев:** Python-репозитории с 10+ звездами, 10+ форками, недавней активностью и open-source лицензиями
2. **Построение маппинга PR-Issue:** Однозначные соответствия между merged pull request'ами и закрытыми issues
3. **Извлечение и фильтрация метаданных:** Метаданные issue и PR парсятся и фильтруются по длине описания
4. **Извлечение и валидация патчей:** Git diffs валидируются, чтобы подтвердить изменения как исходного кода, так и тестов
5. **Валидация сборки репозитория:** Docker-based настройка окружения с проверкой выполнения pytest
6. **End-to-End выполнение задач:** Тестирование в контролируемой среде для воспроизводимости
7. **LLM-based оценка качества:** Модель Qwen3-32B оценивает корректность задачи, корректность тестов, полноту тестов и сложность

### Статистика датасета

- **Масштаб:** Идентифицировано примерно 10,000 потенциальных задач
- **Текущая доступность:** 728 валидированных примеров из 200+ репозиториев
- **Частота обновлений:** Квартальные обновления с новыми реальными задачами
- **Среда выполнения:** Docker-based контейнеры со стандартизированными базовыми образами
- **Контроль качества:** Многоэтапная фильтрация и LLM-based валидация

Весь конвейер реализован как [Python-пакет](https://pypi.org/project/repositorytest) и может быть выполнен для любого GitHub-репозитория, обеспечивая воспроизводимость и расширяемость.

## Ключевые особенности

### Динамические обновления

В отличие от статичных бенчмарков, SWE-MERA обновляется ежеквартально новыми, ранее не встречавшимися задачами. Такой непрерывный цикл обновлений:
- Отражает последние вызовы в разработке программного обеспечения
- Минимизирует риски запоминания решений моделями
- Поддерживает различительную способность бенчмарка со временем
- Позволяет обнаруживать загрязнение данных через временной анализ

### Валидация качества

Каждая задача проходит строгую валидацию:
- **Автоматизированное тестирование:** pytest-based проверка в изолированных Docker-окружениях
- **LLM-based фильтрация:** Задачи из нижнего квартиля отфильтровываются на основе оценок корректности и полноты
- **Реальная релевантность:** Сохраняются только задачи с реальными решениями и полноценными тестами
- **Баланс сложности:** Сохраняются как простые, так и сложные задачи для комплексной оценки

### Детектирование загрязнения данных

[Интерактивный лидерборд](#/leaderboard) оснащен временным слайдером, позволяющим визуализировать метрики оценки в разные временные периоды, выявлять потенциальные события загрязнения в обучающих данных моделей, сравнивать производительность на задачах до и после cutoff и обеспечивать прозрачную воспроизводимую оценку.

## Оценка

SWE-MERA демонстрирует сильную различительную способность на современных моделях. Оценка с использованием агента Aider на дюжине недавних LLM показывает четкую стратификацию производительности. Бенчмарк использует две ключевые метрики: pass@1 (доля успеха с первой попытки) и pass@6 (доля успеха среди шести независимых попыток).

Сравнение год к году выявляет интересные паттерны: некоторые модели показывают различия в производительности между задачами 2024 и 2025 годов, что потенциально указывает на разную степень экспозиции к данным.

## Процесс подачи результатов

Для участия в оценке SWE-MERA:

1. **Загрузите датасет:** Доступен на [Hugging Face](https://huggingface.co/datasets/MERA-evaluation/SWE-MERA)
2. **Запустите агента:** Выполните ваш software engineering агент на датасете
3. **Отправьте результаты:** Создайте pull request в [репозиторий подач](https://github.com/MERA-Evaluation/SWE-MERA-submissions)
4. **Обновление лидерборда:** Валидные подачи появляются в течение двух рабочих дней
